# .claude/ — AI Development Configuration

This directory contains configuration for AI coding tools — primarily [Claude Code](https://docs.anthropic.com/en/docs/claude-code), with cross-tool support via `AGENTS.md`.

## Prerequisites

### Codex CLI (Required for audit commands)

The `cc-suite` plugin (`cc-suite@xiaolai`) uses OpenAI's Codex as an independent second opinion for code audits. The plugin is installed as a Claude Code plugin and registers Codex as a project-level MCP server in `.mcp.json` (use `/cc-suite:init` or `/cc-suite:repair` to add or restore the entry).

Install Codex globally and log in with your subscription:

```bash
npm install -g @openai/codex
codex login                   # Log in with your ChatGPT subscription (recommended)
```

Verify it's on your PATH:

```bash
codex --version
```

> **Prefer subscription auth over API keys.** `codex login` uses your ChatGPT Plus ($20/mo) or Pro ($200/mo) subscription, which is dramatically cheaper than `OPENAI_API_KEY` pay-per-token billing for heavy coding sessions. API keys work as a fallback — see `codex login --with-api-key`.

### Why Codex?

Using a second AI model for code review catches blind spots that a single model might miss. Claude writes the code; Codex audits it independently. This cross-model verification is built into the `/cc-suite:audit` and `/fix-issue` workflows.

## How Codex MCP Works

The `cc-suite@xiaolai` plugin registers the `codex-cli` MCP server in the project's `.mcp.json` (Claude Code MCP namespace `mcp__codex-cli__*`). The server runs in a sandboxed read-only context — it can read the codebase but cannot modify files.

Project-specific Codex settings live in `.cc-suite.md` at the project root (generated by `/cc-suite:init`).

## Directory Structure

```
.claude/
├── README.md              # This file
├── settings.json          # Team-shared settings (checked in)
├── settings.local.json    # Personal settings (gitignored)
├── rules/                 # Auto-loaded project rules
├── commands/              # Project-specific slash commands
├── skills/                # Extended capabilities (planning, Tauri, etc.)
└── agents/                # Subagent definitions for /feature-workflow
```

### Settings Files

| File | Shared? | Purpose |
|------|---------|---------|
| `settings.json` | Yes | Team-wide plugin config (e.g., enabled plugins) |
| `settings.local.json` | **No** (gitignored) | Personal permissions, tool approvals, output style |

`settings.local.json` accumulates permissions as you approve tool calls during sessions. It's machine-specific and should not be committed.

## Bot Cost Reports

Each Claude bot workflow (review, audit, fix) generates a cost report after every run, visible in the Actions run's **Step Summary** tab. The report includes workflow name, event type, model, duration, and status.

Composite action: `.github/actions/claude-report/action.yml`
