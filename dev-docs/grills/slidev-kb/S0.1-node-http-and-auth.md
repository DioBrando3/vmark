# S0.1 — Node HTTP server + ADR-9 cookie auth (server half)

> Result: **PASS (server half)** — Rust spawn of a codesigned in-bundle Node, and
> the macOS quarantine question, are tracked in S0.5 + Phase 1 (external infra).
> Date: 2026-06-25

## Probe

Serve HTTP from Node on `127.0.0.1`, perform the ADR-9 bootstrap→cookie
handshake, prove it works over a real socket from both a programmatic client
(and, by construction, a browser/webview — standard cookies + fetch).

## What was built + evidence

- `src/server/auth.ts` — bootstrap-token → HttpOnly, SameSite=Strict session
  cookie; constant-time compare; bootstrap route self-authenticates, everything
  else 401s without a valid cookie.
- `src/server/runtime.ts` — `@hono/node-server` listen on `127.0.0.1:0`,
  port-file `{port, token}` (mode 0600), watcher wired to SSE reload.
- `src/server/runtime.test.ts` — **live over a real socket** (not just
  `app.request`): binds loopback, writes the port-file, 401 unauthenticated,
  302 + Set-Cookie on bootstrap, 200 authenticated note render over the wire,
  and a watcher-driven `/__health` doc-count change after adding a file.

`pnpm --filter @vmark/content-server test` → 51/51 pass; `pnpm build` (tsc)
clean.

## What this does NOT yet prove (honest residue → Phase 1 / S0.5)

- **Spawning from Rust** via `build_command` + `login_shell_path`. The CLI entry
  (`src/cli.ts`) exists and is the spawn target; the Rust `ContentServerManager`
  is Phase 1.
- **Codesigned in-bundle Node ≥20.12 + macOS Gatekeeper quarantine.** This is the
  central trust question (ADR-2) and requires Apple signing/notarization infra —
  it cannot be PASS/FAIL-ed in a dev shell. Tracked in S0.5 + Phase 1; the leading
  approach remains "ship + codesign the Node binary in-bundle, provision only the
  JS tree."

## Decision recorded

- Cookie transport (not per-request header) is validated end-to-end — confirms
  ADR-9 is the right model and unblocks external-browser + SSE + HMR auth.
- In-app KB surface (Phase 5): with cookie auth proven over the wire, an
  embedded webview pointed at the loopback origin is viable; the native React
  panel will consume the same JSON APIs. Final pick deferred to Phase 5 build.
