# S0.6 — Threat model + header/CSP design (content server)

> Result: **PASS** (design produced; implemented incrementally from Phase 1)
> Date: 2026-06-25

Security is designed up front (review D2.4/D5.3), not deferred to Phase 8.

## Assets

Workspace markdown + rendered HTML; the loopback origin; the bootstrap token +
session cookie; the provisioned runtime bundle.

## Trust boundaries

1. Browser/webview ↔ content server (loopback HTTP).
2. Content server ↔ filesystem (workspace root).
3. VMark/Rust ↔ provisioning download source (network).

## Threats → mitigations

| # | Threat | Mitigation | Where |
|---|--------|-----------|-------|
| T1 | Another local process/app hits the server | Bind `127.0.0.1` only; bootstrap token → HttpOnly cookie; constant-time token compare | `auth.ts`, `runtime.ts` (done) |
| T2 | Token leaks via browser history | Single-use/short-TTL bootstrap token in the `/__auth` redirect; cookie is HttpOnly, never in URLs after | `auth.ts` (bootstrap one-shot; TTL hardening = Phase 4 follow-up) |
| T3 | Path traversal out of workspace | `containedAbsPath` root-prefix check; symlinks skipped in the walker | `createServer.ts`, `walk.ts` (done) |
| T4 | Stored XSS via note content | DOMPurify-on-jsdom; FORBID script/style/iframe + event-handler attrs | `renderMarkdown.ts` (done) |
| T5 | CSRF on state-changing routes | GET routes are side-effect-free; export/mutations are POST + SameSite=Strict cookie (+ double-submit token) | Phase 7 |
| T6 | Remote resource exfiltration / drive-by | CSP per trust state: `default-src 'self'`; `img-src 'self'` (untrusted) / `+https:` (trusted); `script-src 'self'`; `connect-src 'self'` (SSE/HMR); `frame-ancestors 'none'` | Phase 1 header middleware (designed here) |
| T7 | Malicious runtime bundle | CI-built from pinned lockfile; signed manifest verified pre-extraction; native-addon ban | ADR-2 / Phase 1 + S0.5 |
| T8 | Serving an untrusted workspace | `workspace.isWorkspaceTrusted` gate before the server starts | Phase 1 (Rust manager) |

## CSP header (baseline, trusted workspace)

```
Content-Security-Policy:
  default-src 'self';
  img-src 'self' data: https:;
  script-src 'self' 'unsafe-inline';   /* KaTeX/mermaid client bundles; tighten with hashes in Phase 8 */
  style-src 'self' 'unsafe-inline';
  connect-src 'self';
  frame-ancestors 'none';
  base-uri 'none';
  object-src 'none'
```

Untrusted workspace tightens `img-src`/`script-src` to `'self'` and drops
`https:`. `'unsafe-inline'` for script is a Phase-8 hardening target (move to
per-bundle hashes once the client asset set is frozen).

## Residue

- TTL/single-use enforcement on the bootstrap token — Phase 4 hardening.
- CSP `'unsafe-inline'` → hashes — Phase 8.
- Manifest signing key management — Phase 1 + ADR-2 (external infra).
